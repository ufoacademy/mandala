# 연령대별 만성질환 통계 연계 — 설계 문서

날짜: 2026-07-16

## 배경

건강만다라 프리미엄 리포트에 HIRA(건강보험심사평가원) 공공데이터를 연계해서, 그 사람과 같은 연령대·성별에서 실제로 흔한 만성질환을 보여주고, 회색지대 검진수치의 기전 설명에도 "같은 연령대에서 얼마나 흔한 문제인지"를 엮어 동기부여를 강화한다.

## 데이터 출처

- **건강보험심사평가원_질병정보서비스** Open API (data.go.kr, 서비스ID 15119055) — 성별·연령별 질병통계를 실시간 조회 가능한 진짜 REST API.
- 사용자가 data.go.kr에서 활용신청 후 서비스키를 발급받아 `HIRA_API_KEY` 환경변수로 등록 (기존 `ANTHROPIC_API_KEY`, `SHEETS_WEBHOOK_URL`과 동일한 패턴).
- **키가 없거나 API 호출이 실패해도 리포트 생성 자체는 항상 정상 진행** (best-effort, 기존 Sheets 웹훅과 동일한 원칙) — 이 기능이 꺼져 있으면 관련 필드만 조용히 비워짐.

## 아키텍처

1. `api/generate-report.js`에서 사용자의 `age`(나이)·`gender`(성별)를 HIRA API가 쓰는 연령구간(5세/10세 단위) 형식으로 변환.
2. HIRA API를 호출해 해당 연령대·성별의 다빈도 질병 목록을 가져오되, **만성질환 관련 질병으로 미리 정해둔 목록(당뇨병·고혈압·이상지질혈증·지방간·비만·대사증후군 관련 등, KCD 코드 기준)으로만 필터링**해서 그중 가장 흔한 것을 채택 (감기·치과질환 등 무관한 질병이 뽑히지 않도록).
3. 이 결과(`{diseaseName}`)를 `buildPrompt`에 전달 → AI는 이 사실을 인용해 표지용 동기부여 문장(`ageGroupInsight`)을 작성.
4. 같은 데이터를 회색지대 항목과도 매칭 가능하면(예: 고혈압 통계 ↔ 혈압 수치), `grayZoneInsights`의 mechanism 문장 안에 "귀하와 같은 연령대에서도 흔한 문제"라는 언급을 자연스럽게 포함하도록 프롬프트에 지시.
5. HIRA 호출은 리포트 생성 요청마다 실시간으로 이루어짐 (별도 캐싱 없음) — 이미 리포트 생성 자체가 60~120초 걸리므로, 추가 API 호출 1회의 지연은 무시할 만한 수준으로 판단.

## 스키마 변경 (`lib/reportSchema.js`)

- 신규 필드 `ageGroupInsight`: `{ diseaseName: string, motivationalSentence: string } | null` — HIRA 데이터를 못 가져온 경우 `null`.
  - AI가 아니라 **클라이언트/서버가 계산한 diseaseName**(HIRA 응답에서 그대로 가져온 질병명, 숫자 아님)을 프롬프트로 전달하고, AI는 그 질병명을 인용해 `motivationalSentence`(1문장, 예방 동기부여)만 작성 — 기존 "숫자는 계산, 서술은 AI" 원칙과 동일하게 "사실은 외부 데이터, 문장은 AI"로 확장.

## report.html 변경

- 표지(1페이지)에 `positioning-box` 위나 아래에 새 박스 추가: "같은 연령대·성별 예방 신호" 같은 라벨 + `motivationalSentence`. `ageGroupInsight`가 없으면(HIRA 데이터 없음) 이 박스 자체를 렌더링하지 않음 — 페이지 레이아웃이 깨지지 않도록.

## 지금 구현하는 것 / 나중으로 미루는 것

**지금 구현(키 없이도 완성 가능):**
- 나이·성별 → HIRA 연령구간 변환 함수
- 만성질환 필터 목록(KCD 기준) 초안
- 스키마 필드, 프롬프트 지시문, report.html 렌더링
- `fetchAgeGroupDiseaseStats()` 함수 — `HIRA_API_KEY` 없으면 즉시 `null` 반환 (스텁이 아니라 "키 없음"이라는 정상 분기)

**서비스키 발급 후 마무리:**
- `fetchAgeGroupDiseaseStats()` 안에서 실제 HIRA API를 호출하는 부분 — 정확한 요청 파라미터명·응답 필드명은 Swagger 문서/실제 응답을 봐야 확정 가능하므로, 키를 받으면 실제 호출로 1회 테스트 후 파싱 로직을 맞춘다.

## 범위 밖

- 통계 데이터 캐싱/배치 갱신 — 매 요청 실시간 호출로 충분하다고 판단, 필요해지면 추후 별도 작업.
- 다빈도질병통계·국민관심질병통계 등 파일 기반 데이터셋 연동 — 이번 범위는 실시간 Open API(질병정보서비스)만 다룸.
