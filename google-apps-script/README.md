# 리드 저장용 Google Apps Script 배포 방법

1. 리드를 저장할 새 Google Sheets 문서를 만듭니다. 첫 행에 헤더를 넣어둡니다: `날짜 | 이름 | 나이 | 성별 | 이메일 | 연락처 | 총점 | 약점영역1 | 약점영역2 | 약점영역3 | ID`
   - 기존에 이미 리드 시트를 운영 중이었다면, 마지막 칸에 `ID` 헤더 한 칸만 추가하면 됩니다.
2. 같은 스프레드시트에 **빈 탭을 하나 더 만들고 이름을 정확히 `1점문항추적`으로 지정**합니다. 헤더 행은 비워둬도 됩니다 — 스크립트가 첫 실행 시 자동으로 채웁니다.
3. 메뉴에서 **확장 프로그램 → Apps Script**를 클릭합니다.
4. 기본 코드를 지우고 `lead-webhook.gs` 내용을 붙여넣습니다.
5. 우측 상단 **배포 → 새 배포**를 클릭합니다 (기존 배포가 있다면 **배포 → 배포 관리 → 수정**으로 기존 배포를 업데이트해도 됩니다).
6. 유형 선택에서 **웹 앱**을 선택합니다.
7. 설정:
   - 실행 계정: 나(본인 계정)
   - 액세스 권한이 있는 사용자: 모든 사용자
8. **배포**를 클릭하고 권한 승인을 완료합니다.
9. 발급된 **웹 앱 URL**을 복사합니다 (`https://script.google.com/macros/s/XXXX/exec` 형태).
10. 이 URL을 Vercel 프로젝트의 `SHEETS_WEBHOOK_URL` 환경변수로 등록합니다.

## 데이터 흐름

- **진단을 완료하는 모든 사용자** (프리미엄 리포트 신청 여부 무관): 결과 화면이 뜨는 순간 `type:"diagnosis"`로 전송되어 `1점문항추적` 탭에 익명으로 기록됩니다 (ID, 날짜, 총점, 8영역 점수(%), 1점 받은 문항 텍스트 — 1점이 아닌 문항 칸은 빈칸).
- **프리미엄 리포트를 신청한 사용자**: 기존처럼 이름/나이/성별/이메일/연락처/총점/약점영역 top3가 리드 시트(첫 번째 탭)에 기록됩니다. 같은 사람이 진단을 마치고 이어서 프리미엄을 신청했다면, 두 탭에 동일한 `ID`가 기록되어 서로 연결해서 조회할 수 있습니다.

## 동작 확인 (curl)

프리미엄 리드 저장 확인:

```bash
curl -X POST "여기에_배포된_웹앱_URL" \
  -H "Content-Type: application/json" \
  -d '{"type":"premium_lead","id":"test-id-1","timestamp":"2026-07-15T00:00:00.000Z","name":"테스트","age":"35","gender":"f","contact":"010-0000-0000","email":"test@example.com","totalScore":160,"weakArea1":"식 40%"}'
```

익명 진단 기록 확인:

```bash
curl -X POST "여기에_배포된_웹앱_URL" \
  -H "Content-Type: application/json" \
  -d '{"type":"diagnosis","id":"test-id-1","timestamp":"2026-07-15T00:00:00.000Z","totalScore":160,"areaScores":[{"id":1,"pctVal":50},{"id":2,"pctVal":50},{"id":3,"pctVal":50},{"id":4,"pctVal":50},{"id":5,"pctVal":50},{"id":6,"pctVal":50},{"id":7,"pctVal":50},{"id":8,"pctVal":50}],"lowScoreItems":[{"code":"1-A-1","question":"테스트 문항입니다."}]}'
```

각 탭에 새 행이 추가되면 성공입니다 (두 curl 모두 `id`를 `test-id-1`로 동일하게 넣었으니, 두 탭에서 같은 ID로 조회되는지도 확인해보세요).
